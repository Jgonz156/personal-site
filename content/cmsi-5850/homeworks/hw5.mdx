<QuestionBox qid="Q5" title="Reading and Applying Rules" points={15}>

In LN 11, we formalized operational semantics using **inference rules** — premises above the line, conclusion below — and introduced the **Omega (Ω) technique** for reading them: start at the bottom-left (what you have), loop up through the premises (what you must check), and come back down to the bottom-right (what you get).

<SubQuestion qid="Q5a" points={5}>

**The Omega Technique**

For each of the following inference rules, trace the Ω path and explain what the rule does in plain English. Identify each rule's name and state whether it is an **axiom** (no premises) or a **proper rule** (has premises).

**Rule 1:**

<DisplayMath formula="\frac{\sigma(x) = v}{\langle x, \sigma \rangle \Downarrow v}" />

**Rule 2:**

<DisplayMath formula="\frac{\langle e_1, \sigma \rangle \Downarrow v_1 \quad \langle e_2, \sigma \rangle \Downarrow v_2 \quad v_1 \oplus v_2 = v}{\langle e_1 \oplus e_2, \sigma \rangle \Downarrow v}" />

**Rule 3:**

<DisplayMath formula="\frac{\langle e, \sigma \rangle \Downarrow v}{\langle x = e, \sigma, o \rangle \Downarrow \langle \sigma[x \mapsto v], o \rangle}" />

**Rule 4** (novel — not from LN 11):

<DisplayMath formula="\frac{\langle e, \sigma \rangle \Downarrow v \quad v < 0 \quad (-1) \times v = v'}{\langle \textbf{abs}(e), \sigma \rangle \Downarrow v'} \qquad \frac{\langle e, \sigma \rangle \Downarrow v \quad v \geq 0}{\langle \textbf{abs}(e), \sigma \rangle \Downarrow v}" />

</SubQuestion>

<QuestionBox qid="Q6" title="Deriving and Understanding Rules" points={20}>

In LN 11, we showed that inference rules are **mechanically derived** from the tree grammar — every production becomes one or more rules. This question tests your ability to derive new rules, reverse-engineer existing ones, and connect rules to code.

<SubQuestion qid="Q6a" points={6}>

**Ternary Conditional Expression**

Petal currently has an if-else **statement**. Suppose we want to add a ternary conditional **expression**: <InlineMath formula="e_1\; \textbf{?}\; e_2\; \textbf{:}\; e_3" />. Unlike the if-else statement from LN 11, this is an *expression* — it evaluates to a value rather than transforming state.

Follow the 4-step derivation process from LN 11:

1. **Identify the node:** The tree grammar production is <InlineMath formula="e_1\; \textbf{?}\; e_2\; \textbf{:}\; e_3" />. It has three children — all expressions.
2. **Determine the cases:** How does the value of $e_1$ determine the result? How many big-step rules do you need?
3. **Write the big-step rules.** Your rules should have the form <InlineMath formula="\langle e_1\; \textbf{?}\; e_2\; \textbf{:}\; e_3,\; \sigma \rangle \Downarrow v" />. Note the configuration shape — this is an *expression* evaluation (no output stream), unlike the if-else *statement* rules from LN 11.
4. **Write the small-step rules.** How many SOS rules do you need? *(Hint: consider the cases where $e_1$ is still reducing, where $e_1$ is a truthy value, and where $e_1$ is a falsy value.)*

</SubQuestion>

<SubQuestion qid="Q6b" points={6}>

**Do-While Loop**

Suppose we want to add a **do-while** loop to Petal: <InlineMath formula="\textbf{do}\; s\; \textbf{while}\; e" />. Unlike the `while` loop from LN 11, the body executes *before* the condition is checked — the body always runs at least once.

Follow the same 4-step process:

1. **Identify the node:** The tree grammar production is <InlineMath formula="\textbf{do}\; s^+\; \textbf{while}\; e" />. It has one statement-list child (the body) and one expression child (the condition).
2. **Determine the cases:** After the body executes and the condition is evaluated, what are the two possible outcomes? Compare to the `while` rules from LN 11 — what structural difference does "body first, then check" create?
3. **Write the big-step rules.** You may reference the `while` rules from LN 11 side-by-side for comparison:

<DisplayMath formula="\frac{\langle e, \sigma \rangle \Downarrow 0}{\langle \textbf{while}\; e\; \textbf{do}\; s,\; \sigma,\; o \rangle \Downarrow \langle \sigma, o \rangle}" />

<DisplayMath formula="\frac{\langle e, \sigma \rangle \Downarrow v \quad v \neq 0 \quad \langle s, \sigma, o \rangle \Downarrow \langle \sigma', o' \rangle \quad \langle \textbf{while}\; e\; \textbf{do}\; s,\; \sigma',\; o' \rangle \Downarrow \langle \sigma'', o'' \rangle}{\langle \textbf{while}\; e\; \textbf{do}\; s,\; \sigma,\; o \rangle \Downarrow \langle \sigma'', o'' \rangle}" />

4. **Write the small-step unfolding rule.** Recall that the `while` unfolding rule from LN 11 rewrites the loop into a conditional. How does the do-while unfolding differ?

</SubQuestion>

<SubQuestion qid="Q6d" points={4}>

**Rules-to-Code Reasoning**

In LN 11, we showed that every big-step inference rule maps directly to a clause in a recursive interpreter. Consider the **print** rule and its corresponding JavaScript code from LN 11:

**The Rule:**

<DisplayMath formula="\frac{\langle e, \sigma \rangle \Downarrow v}{\langle \textbf{print}\; e, \sigma, o \rangle \Downarrow \langle \sigma, o \cdot v \rangle}" />

  const value = evaluate(node.expression, env);

Now consider the **while** rule and its code:

**The Rule:**

<DisplayMath formula="\frac{\langle e, \sigma \rangle \Downarrow v \quad v \neq 0 \quad \langle s, \sigma, o \rangle \Downarrow \langle \sigma', o' \rangle \quad \langle \textbf{while}\; e\; \textbf{do}\; s,\; \sigma',\; o' \rangle \Downarrow \langle \sigma'', o'' \rangle}{\langle \textbf{while}\; e\; \textbf{do}\; s,\; \sigma,\; o \rangle \Downarrow \langle \sigma'', o'' \rangle}" />

<QuestionBox qid="Q7" title="Proof Trees" points={15}>

In LN 11, we showed that inference rules compose into **proof trees** (derivation trees) — tree-shaped proofs that a specific program produces a specific result. Each node is a judgment, each edge is a rule application, and the leaves are axioms.

This is a **guided exercise**. Each sub-question builds on the previous one. By the end, you will have constructed a complete proof tree for a Petal program with arithmetic, assignment, and control flow.

Consider the following Petal program:

```

if x - 4 {
  y = x * 2;
} else {
  y = 0;
}
print y;
```

**Step 1: The Innermost Expression**

Build the proof tree for evaluating the expression `2 + 3` in the initial empty environment $\sigma_0$. Your tree should show:
- The numeral valuation axioms for `2` and `3` (the leaves)
- The binary operation rule combining them (the root)

<DisplayMath formula="\frac{\displaystyle\frac{}{\langle 3, \sigma \rangle \Downarrow 3} \quad \frac{}{\langle 5, \sigma \rangle \Downarrow 5} \quad 3 + 5 = 8}{\langle 3 + 5, \sigma \rangle \Downarrow 8}" />

</SubQuestion>
