### Game Trees and Minimax

When an AI plays a game like chess or tic-tac-toe, it models the game as a **tree**. Each node is a game state (a board position). Each edge is a possible move. The AI explores this tree to find the best move.

This is literally the graph theory from LN20–23: the game tree is a rooted tree where each node has children corresponding to legal moves. The AI performs a depth-limited search (like DFS from LN20) and evaluates positions using a scoring function.

The **minimax algorithm** assigns scores to game states. One player (X) tries to **maximize** the score; the other (O) tries to **minimize** it. The algorithm propagates scores from leaf nodes upward:

- At a maximizing node: take the **max** of children's scores
- At a minimizing node: take the **min** of children's scores

Click on child boards to explore different move sequences. Watch how the minimax scores propagate upward — X (the maximizer) picks the child with the highest score; O (the minimizer) picks the child with the lowest.

> **Key Insight:** The game tree is a graph (LN20). Minimax is a tree traversal (like DFS). The evaluation function maps game states to numbers — it is a function from boards to integers (LN14–15). AI game-playing is graph theory + functions + search, all concepts from this course.

### Knowledge Representation and Logic

In CMSI 3300, you will learn that AI systems represent knowledge using **logic** (LN1–3) and **graphs**. A knowledge base is a set of logical statements; inference is logical deduction. Constraint satisfaction problems — like the eight circles puzzle from LN22 — are the foundation of AI planning.

Neural networks are **weighted directed graphs** (LN23). Nodes are neurons, edges carry weights, and the network learns by adjusting weights. Bayesian networks are directed acyclic graphs (DAGs) that encode probabilistic relationships — the graph structure determines which variables influence which.

### Pathfinding with A*

Every game with enemies, NPCs, or navigation uses **pathfinding** — finding the shortest route from point A to point B on a map while avoiding obstacles.

The map is a **graph**: each walkable cell is a vertex, adjacent cells share edges, and walls remove edges. Finding the shortest path is exactly Dijkstra's algorithm from LN23, enhanced with a heuristic to search more efficiently.

**A\*** (A-star) is Dijkstra + a heuristic. Instead of exploring all vertices in order of distance from the start (like Dijkstra), A* prioritizes vertices that are *closer to the goal*. The heuristic estimates the remaining distance, guiding the search.

> **Key Insight:** Each grid cell is a vertex. Adjacent cells share edges. Walls remove edges. Pathfinding is **shortest path on a weighted graph** — directly from LN23. The A* heuristic (Manhattan distance) is the optimization that makes it practical for real-time games.

### Procedural Generation and Combinatorics

Procedural content generation — creating random but structured game worlds — relies on combinatorics (LN16–17). How many distinct dungeons can be assembled from $n$ room types with $k$ connections? That is a counting problem. How do you ensure connectivity? Graph theory (LN21). How do you generate balanced encounters? Constraint satisfaction (LN22).

Click **Generate** to create a random dungeon — each click answers a different combinatorial configuration:

Notice how the generator must solve two problems simultaneously: room placement is a **counting** problem (how many valid configurations exist?), while corridor connectivity is a **graph theory** problem (we connect rooms using a spanning tree to guarantee reachability with minimal corridors).

### Network Routing IS Shortest Path

When you send a packet across the internet, it hops from router to router until it reaches its destination. How does each router decide where to send the packet next?

**OSPF (Open Shortest Path First)**, one of the most widely used internet routing protocols, runs Dijkstra's algorithm. Each router maintains a weighted graph of the network, where vertices are routers, edges are links, and weights are link costs. The shortest path from this router to the destination determines the next hop.

### Research Frontiers

Discrete mathematics powers cutting-edge research across many fields:

- **Quantum computing** uses linear algebra over Boolean fields — the same 0/1 algebra from LN5, extended to superposition and entanglement

### Historical Context

- **John von Neumann (1903–1957)** — Co-founded game theory with the minimax theorem (1928); the same von Neumann who gave us the ordinal construction in LN24
- **Peter Hart, Nils Nilsson, Bertram Raphael** — Invented A* at Stanford Research Institute (1968); it remains the gold standard for pathfinding 50+ years later
- **Claude Shannon (1916–2001)** — Proposed the first chess-playing algorithm (1950), building on his Boolean algebra work from LN26
- **Radia Perlman (b. 1951)** — Invented the Spanning Tree Protocol (STP) for Ethernet networks, directly implementing the graph theory spanning tree concept

### Connection to Programming

- Python's `heapq` powers A* and Dijkstra — the same priority queue from LN23
- `networkx` is a Python library for graph algorithms — BFS, DFS, shortest paths, MSTs, all from LN20–23
- Game engines (Unity, Unreal) implement pathfinding using navigation meshes — graph representations of walkable space
- Python's `graphlib.TopologicalSorter` (introduced in 3.9) implements topological sort on DAGs — directly from graph theory
